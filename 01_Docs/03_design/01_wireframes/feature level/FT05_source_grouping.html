FT05 — Source Grouping & Organisation

Feature board · CR-260522-2101-001 · driven by 01_Docs/04_content/images/screenshot.png
Reading guide. Each card below depicts one screen state for the feature. Static visuals only — interaction flows live in journey level/ boards. Colour dots on source rows are the existing FT01 palette (auto-assigned, immutable). Indigo dashed borders mark drop targets during drag.

Annotations appear under each board (§5.1.2: PNGs carry no annotations; HTML boards do).
Note on PNG output: this session authored the boards as self-contained HTML with inline SVG (PNG binary generation is not available to Claude Code). Open in a browser to view the rendered visuals; export to PNG via headless browser if PNG fixtures are needed.

Feature states

Sidebar — tree state (1+ Group exists)FT05_S01_sidebar_tree
Annotations.
  • Top-right `+` (filled): existing FT01 add-source. New `+` (outlined) immediately right of it: new group tooltip.
  • Group rows are bold/larger; subgroup rows indent one level; sources indent two levels (under a subgroup) or one level (direct child of group).
  • Chevrons: ▾ expanded, ▸ collapsed. Click toggles `Group.collapsed` / `Subgroup.collapsed` and persists in DB.
  • "Ungrouped" overflow zone always present at the bottom — even when empty (per AC-260522-2118-010).
  • "+ New group" footer affordance duplicates the header `+` (discoverability) — implemented as the same handler.
Sidebar — flat state (zero Groups, FT03 fallback)FT05_S02_sidebar_flat
Annotations.
  • No chevrons, no indentation hierarchy, no "Ungrouped" zone — FT03's pre-CR sidebar contract is preserved (AC-260522-2117-006).
  • "+ New group" CTA at bottom still rendered — the affordance must be discoverable even before any group exists.
  • Activates the onboarding banner above the feed pane (see Card 3 below).
Onboarding banner — first-run (above feed)FT05_S03_banner
Annotations.
  • Visibility predicate: `settings.grouping_banner_dismissed = 0` AND no Group row exists. Once either side flips, banner stays hidden permanently.
  • Right `×` → PATCH /settings `grouping_banner_dismissed = true`.
  • Auto-dismiss: POST /groups handler sets the flag in the same transaction (AC-260522-2118-030).
Drag-and-drop — source onto subgroup targetFT05_S04_drag
Annotations.
  • Active drop target: indigo dashed border + "drop here" text. Other group/subgroup rows: no border (not currently hovered).
  • Ghost row follows the cursor; original row in Ungrouped goes 30% opacity + strikethrough until the drop commits.
  • On drop → PATCH /sources/{id}/parent with `group_id` (AI) + `subgroup_id` (Newsletters). Invariant satisfied (AC-260522-2118-024).
